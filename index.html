¿Qué demuestra esta página?

Esta página fue creada para mostrar cómo una web puede ser más inclusiva para personas que usan teclado, lector de pantalla o necesitan una interfaz clara.

Aquí se aplican prácticas como textos alternativos en imágenes, formularios con etiquetas, foco visible, navegación sin mouse y atributos ARIA para mejorar la interacción.

1. Uso correcto de alt en imágenes

Las imágenes deben incluir un texto alternativo que describa su contenido o función. Esto ayuda a los lectores de pantalla y a usuarios que no pueden ver la imagen.

2. Formulario accesible con label

Cada campo del formulario tiene una etiqueta asociada. Así, el usuario entiende qué información debe ingresar y puede interactuar mejor con tecnologías de apoyo.

3. Navegación funcional solo con teclado

En esta página puedes moverte usando la tecla Tab, activar botones con Enter o Espacio, y regresar con Shift + Tab.

Además, todos los elementos interactivos muestran un foco visible para que el usuario sepa dónde está posicionado.

4. Uso de atributos ARIA

Los atributos ARIA ayudan a describir mejor algunos componentes para lectores de pantalla, especialmente cuando hay interacción dinámica.

En este botón se usa aria-expanded para indicar si el contenido está abierto o cerrado. También se usa aria-controls para relacionar el botón con el panel que controla.
